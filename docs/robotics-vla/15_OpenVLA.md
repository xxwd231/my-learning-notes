## 一、 模型整体架构与视觉编码器

OpenVLA 是基于 **Prismatic-7B** 视觉语言模型（VLM）微调和扩展而来的具身大模型，其底层大语言模型为 **Llama 2 7B**。

```
[ Input Image ] ──► DinoV2 (空间特征: H,W,C1) ┐
                 ──► SigLIP  (语义特征: H,W,C2) ┴─► Concat (H,W,C1+C2) ──► MLP Projector ┐
                                                                                           ├─► Llama 2 7B ──► Action De-Tokenizer ──► 7D Action
[ Instruction ] ─────────────────────────────────► Llama Tokenizer ──────────────────────┘

```

### 1. 核心组件构成

-   **语言编码器 (Language Encoder)**：使用 **Llama Tokenizer**，负责将输入的自然语言指令（如 _"Put eggplant in bowl"_）转化为 Token Embedding 序列。
    
-   **视觉编码器 (Vision Encoder - 双路并行)**：
    
    -   **DinoV2**：负责提取图像的**低级空间几何特征**（如纹理、具体位置、边缘与形状等）。
        
    -   **SigLIP**：负责提取图像的**高级语义特征**（如类别、颜色等）。
        
    -   **特征拼接 (Concatenation)**：两者的特征向量按通道维度拼接，即 $(H, W, C_1) + (H, W, C_2) \to (H, W, C_1+C_2)$。
        
-   **投影层 (MLP Projector)**：将拼接后的视觉特征压缩并投影到 Llama 2 能够理解的“视觉 Token”特征空间。
    
-   **大语言模型 (LLM Backbone)**：使用 **Llama 2 7B**，统一理解输入的视觉与语言特征，并生成对应的动作序列 Token。
    

> **为什么具身机器人必须引入 DinoV2？**
> 
> -   **CLIP/SigLIP 的局限**：擅长“高级语义”，能识别出“这是杯子”，但对具体物体的 3D 深度、边缘和姿态等空间物理细节模糊。
>     
> -   **DinoV2 的优势**：基于自监督学习（如图像块掩码预测），专注于图像的几何结构。
>     
> -   **精髓总结**：_“CLIP 让你知道抓什么，DINO 让你知道怎么抓”_。两者结合显著提高了 LLM 的空间推理与控制能力。
>     

## 二、 动作序列编解码机制 (Action Discretization)

为了让大语言模型能够像生成文本一样直接输出物理控制量，OpenVLA 借鉴了 **RT-2** 的离散化（Discretization）思想，将连续的物理动作空间映射为离散 Token。

### 1. 动作离散化 (Encoding - 连续转离散)

-   **百分位截断 $[P_1, P_{99}]$**：由于机器人传感器数据存在高噪点（离群值），因此保留第 1 到第 99 百分位的数据，避免极值压缩正常动作区间的精度。
    
-   **256 箱子分箱 (256 Bins)**：将 $[P_1, P_{99}]$ 区间均匀切分为 256 个小区间，离散为箱子索引 $\{0, 1, \dots, 255\}$。
    
    $$\text{Bin Width} = \frac{P_{99} - P_1}{256}$$
    

### 2. 动作解码 (Decoding - 离散转连续)

-   **输出维度**：输出 7 维机器人动作（平移 $\Delta x, \Delta y, \Delta z$、旋转 $\Delta \theta_x, \Delta \theta_y, \Delta \theta_z$ 及夹爪开合度 $\Delta \text{Grip}$）。
    
-   **取中心值还原**：模型预测出 Token 索引后，查表找出该索引对应的箱子范围，并取**箱子中心值**作为最终发送给电机的连续控制参数。
    

### 3. 工程技巧：词表覆写 (Token Overwriting)

-   **问题**：强行新增 256 个特殊 Token 会破坏 Llama 2 原生的矩阵维度，破坏预训练权重。
    
-   **解决方案**：Llama 2 词表容量为 32,000。作者直接**覆写了词表末尾使用频率极低（最冷门）的 256 个 Token**。大模型本质上依然在做“下一个 Token 预测”，但被覆写的 Token 实际上代表了电机的物理动作。
    

## 三、 训练策略与数据集筛选

基于 **OpenX**（超过 2M 机器人轨迹）数据集进行微调筛选：

1.  **输入/输出一致性筛选**：仅保留**第三人称视角摄像头**且为**单臂末端执行器**控制的操作数据。
    
2.  **权重平衡与降权**：采用 Octo 的数据混合权重策略，对多样性较低的数据集降权或剔除，提高高多样性数据集的权重。
    
3.  **异常过滤**：过滤掉零值动作等无效或异常数据。
    
4.  **训练资源**：使用 64 张 NVIDIA A100 GPU，Batch Size 设置为 2048，训练用时 14 天。
    

## 四、 参数高效微调 (PEFT) 实验对比

在将 OpenVLA 迁移部署到新机器人配置或新任务时，研究对多种微调策略进行了实验对比：

-   **Full FT (全参数微调)**
    
    -   **成功率**：69.7 ± 7.2%
        
    -   **可训练参数量**：7,188.1 $\times 10^6$
        
    -   **显存占用 (Batch 16)**：163.3 GB*
        
-   **Last layer only (仅微调最后一层)**
    
    -   **成功率**：30.3 ± 6.1%
        
    -   **可训练参数量**：465.1 $\times 10^6$
        
    -   **显存占用 (Batch 16)**：51.4 GB
        
-   **Frozen vision (冻结视觉编码器)**
    
    -   **成功率**：47.0 ± 6.9%
        
    -   **可训练参数量**：6,760.4 $\times 10^6$
        
    -   **显存占用 (Batch 16)**：156.2 GB*
        
-   **Sandwich (三明治微调策略)**
    
    -   **成功率**：62.1 ± 7.9%
        
    -   **可训练参数量**：914.2 $\times 10^6$
        
    -   **显存占用 (Batch 16)**：64.0 GB
        
-   **LoRA (rank = 32)**
    
    -   **成功率**：68.2 ± 7.5%
        
    -   **可训练参数量**：97.6 $\times 10^6$
        
    -   **显存占用 (Batch 16)**：59.7 GB
        
-   **LoRA (rank = 64)**
    
    -   **成功率**：68.2 ± 7.8%
        
    -   **可训练参数量**：195.2 $\times 10^6$
        
    -   **显存占用 (Batch 16)**：60.5 GB
        

> **关键实验结论**：
> 
> 使用 **LoRA ($r=32$)** 微调，只需**单张 A100 GPU** 即可在 10-15 小时内将 OpenVLA 成功适配至新任务。在成功率与全参数微调几乎持平（68.2% vs 69.7%）的前提下，计算量降低了近 **8 倍**，且大幅节省了显存占用。
