## 一、 RT-1 模型基本输入输出

RT-1 是由 Google 推出的基于 Transformer 架构的具身智能动作控制模型，能够将多帧连续的图像和自然语言指令直接映射为离散化控制动作。

```
输入: [6 帧摄像头历史图像] + [自然语言任务指令]
                         │
                         ▼
                     [ RT-1 ]
                         │
                         ▼
输出: [离散化动作 Token 序列 (11 个动作维度)]

```

-   **输入（Inputs）**：
    
    -   **历史图像序列**：来自机器人摄像头的历史图像短序列（通常为历史 6 帧图像，分辨率为 $300 \times 300$）。
        
    -   **任务指令**：通过自然语言描述的具体操作任务（如 "pick up the green can"）。
        
-   **输出（Outputs）**：
    
    -   **Tokenized Actions**：经过离散化标记后的动作指令序列。
        

## 二、 多模态 Tokenization 机制

为了将视觉、语言和动作接入统一的 Transformer 架构，RT-1 对三种模态分别进行了量化与 Token 提取：

```
[6帧 300x300 图像] ---> EfficientNet-B3 ---> 6 x (9x9x512) 特征图
                                                     │
[自然语言指令]     ---> FiLM conditioning modulation   │
                                                     ▼
                            Flatten: 6 x 81 个 Visual Tokens
                                                     │
                                               TokenLearner
                                                     ▼
                            压缩为: 6 x 8 = 48 个 Visual Tokens

```

### 1. 图像标记化（Image Tokenization）

-   **特征提取**：采用在 ImageNet 上预训练的 **EfficientNet-B3** 作为 Backbone。输入 6 帧 $300 \times 300$ 的图像，提取出形状为 $9 \times 9 \times 512$ 的空间特征图。
    
-   **初始展平**：将特征图展平后，每帧图像初始会转换为 **81 个** 视觉 Token（每个维度为 512）。
    
-   **TokenLearner 极简压缩**：通过 TokenLearner 模块将每帧图像的 81 个 Token 自适应压缩为 **8 个** Token。
    
-   **时间维度堆叠**：最终 6 帧图像形成 $6 \times 8 = 48$ 个视觉 Token，并在时间维度上进行堆叠。该设计既保留了丰富的时序运动信息，又充分利用了预训练模型的特征提取能力。
    

### 2. 自然语言指令嵌入（Natural Language Instructions Embedding）

-   **Universal Sentence Encoder (USE)**：基于 Transformer 编码器架构的句子级特征提取器。将输入的任务文本转化为固定维度的语义向量（一句话对应一个任务信号向量），捕获全局上下文语义。
    
-   **池化方式**：USE 在 Transformer Encoder 输出的所有词元向量上进行元素级别求和（Element-wise Sum）得到全局向量。相比 BERT（常用的 `[CLS]` 位置向量）或 DAN（Deep Averaging Network，将 Word2Vec 嵌入后取平均），USE 具备更优的文本长句特征整合能力。
    

### 3. 动作标记化（Action Tokenization）

RT-1 定义了一个 **11 维度的动作向量**，用以全面控制机器人的运动：

```
11维动作向量 = 7个机械臂变量 + 3个底盘运动变量 + 1个模式切换变量

```

-   **机械臂运动（7 个变量）**：
    
    -   **末端位置 (x, y, z)**：笛卡尔坐标系下的空间三维位置。
        
    -   **末端姿态 (Roll, Pitch, Yaw)**：绕 x, y, z 轴的旋转角度。
        
    -   **夹爪开合 (Gripper Opening)**：控制夹爪间距（从 0 全闭到 1 全开）。
        
-   **底盘运动（3 个变量）**：
    
    -   **运动变量**：平移与旋转增量 $(x, y, \text{Yaw})$。
        
-   **模式切换（1 个变量）**：
    
    -   **模式变量**：离散开关，用于在“控制机械臂”、“控制底盘”和“停止”三种模式间进行切换。
        
-   **量化与分类任务转化**：
    
    -   每个变量在其连续的取值范围内被离散均匀化切割为 **256 个区间（Bins）**。
        
    -   动作量化值采用相邻两个时间点的动作增量（Delta）表示。
        
    -   **将连续控制问题转化为 256 分类的交叉熵优化问题**。
        

## 三、 关键架构组件：FiLM 与 TokenLearner

```
   [语言指令向量] 
         │
         ▼
[ identity-initialized FiLM Layer ] ──> (微调 EfficientNet 卷积特征: γ*V + β)
                                                     │
                                                     ▼
                                          [ TokenLearner Module ] ──> (81 -> 8 Tokens)

```

### 1. 视觉与语言融合：FiLM conditioning layer

-   **原理**：特征层线性调制（Feature-wise Linear Modulation）。通过语言向量计算缩放参数 $\gamma$ 和偏置参数 $\beta$，对视觉特征图 $V$ 进行逐通道特征矫正：
    
    $$\text{FiLM}(V) = \gamma \cdot V + \beta$$
    
-   **恒等初始化（Identity-initialized FiLM）**：
    
    -   将权重设为 $0$，$\beta$ 设为 $0$，使得初始阶段 $\gamma=1, \beta=0$。
        
    -   初始时刻纯粹以视觉特征作为输入，随后随着训练逐步融合语言指令，极大地增强了 EfficientNet 从头训练时的稳定性（与 ControlNet 的零卷积 Zero Convolution 思想类似）。
        

### 2. 特征压缩：TokenLearner

-   **原理**：一种元素级别的注意力机制模块（Element-wise Attention）。
    
-   **作用**：自适应地选择和重新组合输入特征图的特征，将 pre-trained FiLM-EfficientNet 输出的 81 个视觉 Token 降采样并重构为 **8 个** 高度浓缩的 Token。
    
-   **集成**：作为 Transformer 模块的预处理部分，参数跟随整个 RT-1 端到端更新。
    

## 四、 Transformer 解码器与注意力机制

RT-1 采用**因果自注意力机制的 Transformer 解码器（Causal Transformer Decoder）**，进行自回归式（Autoregressive）动作生成。

```
输入序列格式: [I_1, I_2, ..., I_48, L, A_1, A_2, ..., A_11]

```

```
           [ 注意力掩码矩阵 (Attention Mask Structure) ]
┌───────────────────────────┬───────────────────────────┐
│ 左上角: 视觉/语言 -> 视觉/语言│ 右上角: 视觉/语言 -> 动作 │
│ (双向注意力 Bi-directional) │ (可关注所有视觉/语言输入) │
├───────────────────────────┼───────────────────────────┤
│ 左下角: 动作 -> 视觉/语言  │ 右下角: 动作 -> 动作      │
│ (可关注所有视觉/语言输入)   │ (因果注意力 Causal Mask)   │
└───────────────────────────┴───────────────────────────┘

```

### 特殊注意力掩码（Attention Mask）设计

为了保证多模态交互特征的充分提取与动作自回归生成的因果性，RT-1 设计了包含四个子区域的特殊注意力掩码矩阵：

-   **左上角区域（视觉/语言 $\rightarrow$ 视觉/语言）**：使用**双向注意力（Bi-directional Attention）**，允许所有视觉 Token 和语言 Token 之间相互 Attention，充分融合多模态全局信息。
    
-   **右下角区域（动作 $\rightarrow$ 动作）**：使用**因果注意力（Causal Attention）**，保证生成当前动作 Token 时只能关注到前面已经生成的动作 Token，遵循时序因果律。
    
-   **右上角区域（视觉/语言 $\rightarrow$ 动作）**：允许在生成任何一个动作 Token 时，全盘关注并引用所有的视觉与语言输入 Token。
    
-   **左下角区域（动作 $\rightarrow$ 视觉/语言）**：允许动作 Token 向前关注所有的视觉与语言上下文特征。
    

## 五、 损失函数与模型训练

由于 RT-1 将动作空间量化成了 256 个离散 Bin，模型训练转化为了标准的分类预测任务：

-   **损失函数（Loss Function）**：使用标准的**多类别交叉熵损失函数（Multi-class Cross-Entropy Loss）**。
    
-   **目标计算**：对于生成的 11 个动作维度，分别计算模型预测的概率分布与真实动作（Ground Truth Delta Bin）之间的交叉熵，并求和优化。
