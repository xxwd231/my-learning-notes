## 一、 泛化挑战：封闭世界 vs 开放世界

从实验室走向真实家庭环境，具身智能面临三个层面的泛化挑战：

-   **封闭世界（实验室）**[cite: 9]
    
    -   **环境**：固定实验室[cite: 9]。
        
    -   **物品**：训练集中常见物品[cite: 9]。
        
    -   **任务指令**：预定义规范指令[cite: 9]。
        
    -   **现状**：现有方法基本可解决[cite: 9]。
        
-   **开放世界（真实家庭）**[cite: 9]
    
    -   **环境**：任意未知的全新家庭[cite: 9]。
        
    -   **物品**：可能从未见过的海量离群物品[cite: 9]。
        
    -   **任务指令**：自然语言，语义极其多样且模糊[cite: 9]。
        
    -   **现状**：必须依赖全新架构与数据策略[cite: 9]。
        
-   **泛化需同时发生的三个层面**[cite: 9]
    
    -   **低层（物理）**：如何抓起这个从未见过的特定勺子？[cite: 9]
        
    -   **视觉**：桌上这堆脏碗碟该怎么处理？[cite: 9]
        
    -   **语义**：“整理厨房”意味着什么，应该做哪些步骤？[cite: 9]
        

## 二、 $\pi_{0.5}$ 的三大核心贡献

-   **1. 开放世界泛化**[cite: 9]
    
    -   在完全陌生的家庭环境中完成复杂的家务任务[cite: 9]。
        
    -   仅基于 **104 个训练家庭**，性能即接近直接在测试家庭微调训练的上限[cite: 9]。
        
-   **2. 异构数据 Co-training 方案**[cite: 9]
    
    -   融合移动机械臂、静态机器人、跨平台数据、网络多模态数据与口头指导数据[cite: 9]。
        
    -   **97.6%** 的预训练数据来自非移动机械臂来源，极大地拓宽了模型的认知泛化边界[cite: 9]。
        
-   **3. 统一分层架构**[cite: 9]
    
    -   高层文本生成 + 低层 Flow Matching，集成在同一个模型中[cite: 9]。
        
    -   联合损失函数：$\mathcal{L} = \mathcal{L}_{CE} + \alpha \cdot \mathcal{L}_{FM}$（预训练时 $\alpha=0$，后训练阶段 $\alpha=10$）[cite: 9]。
        

## 三、 异构数据联合训练策略 (Co-training)

如何让机器人“见多识广”？答案不只是采集更多机器人硬件数据，而是跨模态吸收数据[cite: 9]：

-   **数据融合类型**[cite: 9]：
    
    -   移动机械臂操作数据[cite: 9]。
        
    -   多环境静态机器人数据[cite: 9]。
        
    -   跨机器人平台数据[cite: 9]。
        
    -   互联网多模态海量数据[cite: 9]。
        
    -   口头指导演示数据[cite: 9]。
        
-   **认知逻辑**[cite: 9]：
    
    -   即使模型从未在“客人的厨房”里训练过，但通过在客厅、餐厅以及互联网视频中学习过的操作模式，依然能够泛化到新场景[cite: 9]。
        

## 四、 统一分层控制架构与训练损失

$\pi_{0.5}$ 采用了 **同一模型承担高层与低层** 的两级推理机制[cite: 9]：

### 1. 高层推理：语义子任务预测[cite: 9]

-   **本质**：自回归文本生成（类似 CoT 链式思维，模型先“自言自语”接下来做什么）[cite: 9]。
    
-   **输入**：当前图像 + 任务指令（例：`"clean the bedroom"`）[cite: 9]。
    
-   **输出**：离散文本 Token，指示下一步语义子任务（例：`"pick up pillow"`）[cite: 9]。
    
-   **优势**：文字是语义理解最自然的载体，可直接利用预训练 VLM 的推理能力[cite: 9]。
    

### 2. 低层控制：Flow Matching 动作生成[cite: 9]

-   **本质**：继承自 $\pi_0$ 的连续动作流匹配，由 300M Action Expert 承载[cite: 9]。
    
-   **输入**：当前图像 + 任务指令 + **子任务文本 Token** + 关节状态[cite: 9]。
    
-   **输出**：50 步连续关节角度轨迹序列（约 1 秒动作块）[cite: 9]。
    
-   **损失函数**[cite: 9]：
    
    $$\mathcal{L}_{low} = \mathbb{E}_\tau \left\Vert{} v_\theta(A_t^\tau, \tau, o_t) - (A_t - \epsilon) \right\Vert{}^2$$
    

### 3. 联合损失与两阶段训练[cite: 9]

-   **联合损失函数**[cite: 9]：
    
    $$\mathcal{L} = \mathcal{L}_{CE} + \alpha \cdot \mathcal{L}_{FM}$$
    
    其中 $\mathcal{L}_{CE}$ 为高层文本交叉熵损失，$\mathcal{L}_{FM}$ 为低层 Flow Matching 损失[cite: 9]。
    
-   **训练阶段划分**[cite: 9]：
    
    -   **预训练阶段（上文化课）**[cite: 9]：280k 步，$\alpha = 0$，纯离散 Token 预测（Next-token prediction），只训练大模型的语义大脑[cite: 9]。
        
    -   **后训练阶段（上操场）**[cite: 9]：80k 步，$\alpha = 10.0$，高层与低层联合优化，引入 300M Action Expert 训练物理控制[cite: 9]。
        

## 五、 $\pi_0$ vs $\pi_{0.5}$ 进化对比（列表版）

### 1. 架构基础

-   **$\pi_0$**：PaliGemma VLM + Action Expert[cite: 9]
    
-   **$\pi_{0.5}$**：相同架构（完全继承）[cite: 9]
    

### 2. 动作输出

-   **$\pi_0$**：Flow Matching 连续动作[cite: 9]
    
-   **$\pi_{0.5}$**：相同（完全继承）[cite: 9]
    

### 3. 高层推理

-   **$\pi_0$**：无[cite: 9]
    
-   **$\pi_{0.5}$**：**有（自回归文本 Token，思维链 CoT）**[cite: 9]
    

### 4. 训练数据

-   **$\pi_0$**：多机器人同类数据[cite: 9]
    
-   **$\pi_{0.5}$**：**多来源异构数据 (Co-training)**[cite: 9]
    

### 5. 泛化目标

-   **$\pi_0$**：多任务、多平台[cite: 9]
    
-   **$\pi_{0.5}$**：**开放世界新环境**[cite: 9]
    

### 6. 测试场景

-   **$\pi_0$**：与训练集高度重叠[cite: 9]
    
-   **$\pi_{0.5}$**：**完全陌生的真实家庭**[cite: 9]
    

## 六、 $\pi_{0.5}$ 与 Hi Robot 的继承与区别

-   **理念继承**[cite: 9]：
    
    -   两者均采用“高层语义决策 + 低层动作执行”的分层控制思路[cite: 9]。
        
-   **核心区别**[cite: 9]：
    
    -   **Hi Robot**：高层 VLM 和低层策略是**两个独立的模型/系统**，中间严格依赖自然语言接口交互[cite: 9]。
        
    -   **$\pi_{0.5}$**：**同一个模型权重**同时承担高层和低层，高层推理是模型的“内部自言自语”，训练时端到端联合优化，无需两个独立系统间的通信接口[cite: 9]。
