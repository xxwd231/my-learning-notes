## 一、 核心洞察：多样化上下文与数据价值最大化

$\pi_{0.7}$ 的核心突破在于不只告诉机器人“做什么”，还要通过丰富上下文告诉它“怎么做”：

-   **传统训练模式**：
    
    -   输入单一任务指令（例：`"pick up the cup"`） $\to$ 直接输出动作序列。
        
-   **$\pi_{0.7}$ 训练模式**：
    
    -   **任务指令**：高层任务（例：`"pick up the oven mitt"`）。
        
    -   **子任务指令**：具体执行方式（例：`"grasp the handle with the left hand"`）。
        
    -   **元数据 (Metadata)**：质量、速度与错误标记（例：`Quality: 5/5 | Speed: 2000 | Mistake: false`）。
        
    -   **控制模态**：关节控制 / 末端执行器控制（`joint control / end-effector control`）。
        
    -   **视觉子目标**：当前子步骤结束时的目标图像帧。
        
-   **为什么这种方式有效？**
    
    -   **数据高倍复用**：同一段机器人采集数据可以从不同角度贴“打标签”，大幅提升数据利用率。
        
    -   **次优数据变废为宝**：过去被视为噪点的失败或低速数据，通过元数据标注后成为有价值的负面/控制样本。
        
    -   **测试时灵活引导**：推理阶段可以通过组合不同的 Prompt 元素，精准引导模型输出预期行为。
        

## 二、 架构概览：三大组件与两级推理系统

$\pi_{0.7}$ 模型共有约 **5B 参数**，由三个核心模块构成，并配合两级系统运行：

-   **三大核心组件**：
    
    -   **VLM 骨干 (VLM Backbone)**：4B 参数，基于 Gemma 3，用于深度理解视觉与语言信息。
        
    -   **视频编码器 (MEM)**：包含 400M 视觉编码器，对历史视觉帧进行时空压缩，输出固定数量的 Token。
        
    -   **流匹配动作专家 (Flow Matching Action Expert)**：860M 参数，每次预测 50 步动作 Chunk。
        
-   **推理时的两级系统**：
    
    -   **高层策略**：同架构的 VLA 模型，自动实时生成子任务语言指令。
        
    -   **世界模型**：基于 BAGEL (14B) 微调的模型，生成下一步的视觉子目标图像。
        

## 三、 多模态 Prompt 的五个维度与 Dropout 机制

### 1. Prompt 的五个维度

-   **Observation**：多帧历史观测图像。
    
-   **Task Instruction**：整体任务的语言描述。
    
-   **Subtask Instruction**：当前子步骤的语言描述。
    
-   **Subgoal Image**：当前子步骤结束时的目标画面。
    
-   **Metadata**：Quality / Speed / Mistake 等控制标记。
    

### 2. 关键设计：Dropout 机制

-   **训练机制**：训练时每个组件（指令、子目标图像、元数据等）会被随机丢弃（Dropout）。
    
-   **推理优势**：使得模型在测试阶段可以根据输入条件任意灵活组合：
    
    -   仅提供语言指令时，模型仍可正常工作。
        
    -   叠加子目标图像时，控制性能进一步提升。
        
    -   叠加元数据时，可精确控制机器人的动作质量与运行速度。
        

## 四、 世界模型：视觉子目标生成机制

世界模型的核心思想是“用图像直接告诉机器人这步结束时应该长什么样”：

-   **目标函数**：
    
    $$\max_{\psi} \mathbb{E}_{\mathcal{D}_g} \left[ \mathcal{L}_{\text{CFM}}\left(g_t^*, g_\psi(o_t, \hat{l}_t, m)\right) \right]$$
    
    -   $g_t^*$：训练数据片段末帧（真实的子目标图像）。
        
    -   $o_t$：当前时刻的视觉观测。
        
    -   $\hat{l}_t$：当前步骤的子任务指令。
        
    -   $m$：元数据条件。
        
-   **世界模型的关键特点**：
    
    -   **基础架构**：基于 BAGEL（14B MoT 图像生成模型）进行微调。
        
    -   **物理先验**：融合 Web 海量数据与人类视频增强，获取丰富的物理世界先验知识。
        
    -   **混合数据训练**：训练时混合使用真实未来帧（25% 末帧 + 75% 往后 0~4 秒随机帧）与模型生成的预测帧。
        
    -   **空间精度**：比纯文本语言指令能更精准地刻画复杂空间物理关系，显著增强开放世界的泛化性能。
